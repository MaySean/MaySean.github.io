安全性注意事项

默认情况下，Docker 远程连接是不安全的，因为它没有启用任何身份验证或加密。为了提高安全性，建议使用 TLS 来保护 Docker 远程连接。

总结

通过修改 Docker 的 systemd 服务文件，你可以启用 Docker 的远程连接。请务必注意，开启远程连接可能会带来安全风险，因此建议在生产环境中使用 TLS 进行加密和认证。

根据提供的知识源，我没有足够的信息来确定 Docker 在启动时是否一定会扫描 /etc/systemd/system/docker.service.d/ 目录下的 .conf 文件。然而，我可以提供一些相关的信息：

  1. systemd 配置：
    在使用 systemd 的系统中，/etc/systemd/system/docker.service.d/ 目录确实用于存放 Docker 服务的额外配置文件。这些文件通常用于覆盖或扩展默认的 Docker 服务配置。Engine > Daemon > Troubleshooting the Docker daemon > Daemon > Troubleshoot conflicts between the daemon.json and startup scripts > Configure the daemon host with systemd

  2. 配置加载：
    当使用 systemd 时，这些配置文件应该在 Docker 服务启动时被加载。但是，这个过程是由 systemd 管理的，而不是直接由 Docker 守护进程管理。

  3. 配置生效：
    为了确保这些配置文件被正确加载和应用，通常需要执行以下步骤：

    Engine > Daemon > Daemon proxy configuration > Environment variables

  4. 配置验证：
    可以使用以下命令来验证配置是否被正确加载：

    Engine > Daemon > Daemon proxy configuration > Environment variables

  5. 其他配置方法：
    Docker 还支持其他配置方法，如使用 daemon.json 文件或命令行参数。这些方法可能会与 systemd 配置文件交互或覆盖它们。Engine > Docker daemon configuration overview > Configure the Docker daemon

总的来说，虽然 /etc/systemd/system/docker.service.d/ 目录下的 .conf 文件应该在 Docker 服务启动时被考虑，但具体的行为可能取决于系统配置和 Docker 版本。为了确保配置生效，最好遵循官方文档中的步骤，包括重新加载 systemd 配置和重启 Docker 服务。

如果您遇到特定的配置问题，可能需要查看 Docker 日志或联系 Docker 支持以获取更详细的帮助。
